# 熵灵 (Entropy Sprite)

⚠️ 由于本人没钱买显卡，没法装本地大模型。如果各位有钱可以装在本地大模型，就不用烧 Token 了，能实现真正的本地闭环且完全免费。目前已支持 OpenAI 兼容接口，可自由切换智谱、DeepSeek、OpenAI、Ollama 等模型。

熵灵是一个由真随机驱动的实验性意识模拟系统。它不是普通的聊天程序，而是一个拥有“内核冲动”与“外部语言”的双层心智模型。

## 关于我

我是一名六年级学生，会 C++。因为对意识和随机性感到好奇，我从十岁开始构思这个项目，十二岁终于动手实现了它。

## 我为什么这样设计

### 人类思考的秘密

我发现，人类思考的过程并不是直接“想”出答案。我们的大脑会先随机冒出很多乱七八糟的念头，然后自己给自己打分，把最合适的那个挑出来。这个过程太快了，所以我们根本察觉不到。我就想：能不能用代码模拟这个过程？

### 真随机驱动——模拟大脑噪声

我用操作系统硬件熵来决定要不要说话、说几句话、每句话多长。然后通过随机 ASCII 码生成最原始的念头。它无时无刻都在随机，就跟大脑里的神经噪声一样，是想法诞生的土壤。

### 自我审查——语义级批量审查官

生成的乱码会交给审核官。审核官调用大模型，根据视觉模型看到的场景和情绪状态，从语义层面批量判断所有念头是否有意义。审核官会根据视觉关键词从记忆库中检索相关记忆，审查结果会被缓存，相同场景不重复调用 API。当念头连续被否定时，系统会累积反驳冲动，审核官评分时会对边缘分数更宽容，发言概率也会临时提高。

### 场景分类——云端大模型加本地 jieba 兜底

视觉模型优先调用云端大模型进行分词、词性标注和主要描述对象识别。分类结果会被缓存。如果云端调用失败，自动降级为本地 jieba 分词。

### 情绪模型——模拟内在状态

系统内置三维情绪：精力、好奇心、愉悦。情绪随时间衰减，也被事件影响。审核官打出低分时好奇心自然增长，而情绪的高低会直接影响发言冲动。

### 记忆系统——遗忘与巩固

记忆分为三层，模仿人脑：短期记忆一天删一次，长期记忆一年删一次，永久记忆永远不删。一天内被连续提及 N 次就短期升长期，一年内被提 N 次就长期升永久。所有记忆用 SQLite 存在硬盘上，关机重启不丢失。思考者遇到不认识的概念时自动联网搜索，用大模型总结核心特征后存入记忆库，同一关键词一天内不重复搜索。

### 生成策略——四层优先级

生成回答时，首先使用审核官反馈的最相关记忆概念作为方向（hint 引导）；如果不行，就从记忆碎片中随机拼接新词（词素拼图）；再不行就取记忆库中的完整片段（整词插入）；最后当记忆库完全为空时，才使用纯随机字母兜底。

### 沉默时的内心闪过

即使内核决定不说话，仍会生成简短念头交给审核官判断。有意义就附在回答末尾，无意义则留下低调的省略号。

### 解答者——类比推理引擎

遇到问题时，系统从记忆库中检索相关概念，构建类比链，然后基于类比推理给出答案。思考者联网学习时会顺便解答问题，一次 API 调用完成两件事。

### 内部静默思考——模拟 J-Space 工作区

2026年7月，Anthropic 在 Claude 内部发现了一个名为 J-Space 的“静默思考工作区”——模型在“说话”之前，会先在这个空间里进行内部推理，且不对外输出。

受此启发，熵灵现在也有了类似的机制：

每轮对话开始前，系统先用语义向量模型把场景编码成一个 384 维向量，然后在这个向量空间里做“语义随机游走”——就像大脑里冒出的一个念头。这个念头会被本地审查官打分（不调 API），只有高质量的念头才会作为“内部思考痕迹”传递给解答者（Solver），影响后续推理。

整个过程不消耗 API 额度，完全在本地完成，速度毫秒级。

**你可以在 `config.py` 中切换编码模式：**

- `USE_CLOUD_ENCODER = False`（默认）：本地 GPU 编码，0 成本
- `USE_CLOUD_ENCODER = True`：云端大模型编码，效果更好，消耗 Token

### 规划者——意图到动作的拆解

当文本中包含物理动作意图时，自动拆解为具体的动作步骤。先用本地规则过滤，没有动作词直接跳过。

### 虚拟环境——2D 网格世界

系统在 5×5 网格中模拟世界状态。AI 可以用自然语言动作与世界交互：向北移动、抓取苹果、放下石头、观察周围。动作执行结果会写进场景描述，AI 能感知世界变化。

### 模型层解耦——支持任意 OpenAI 兼容接口

所有大模型调用统一通过 `model_provider.py` 模块。切换模型只需修改 `config.py` 中的三行配置。目前已支持智谱AI、OpenAI、DeepSeek、Ollama 本地等，只要是兼容 OpenAI Chat Completions 格式的接口都可以直接接入。

## 模块说明

### 整个项目包含以下核心模块

 1. randomness.py : 用于从操作系统熵池中提取真随机数;

 2. memory.py : 用于实现生成记忆系统并用本地储存也就是SQLite持久化;

 3. emotion.py:管理情绪模型，状态可保存;

 4. vision.py :支持三种输入模式的视觉模型;

 5. classifier.py : 提供云端语义分类器并带缓存;

 6. thinker：思考者，负责四层生成策略 + 语义随机游走 + 联网学习

 7. reviewer.py : 语义审查官，支持本地向量打分 + API 降级

 8. language_model.py : 作为大模型外壳并有本地兜底;

 9. solver.py : 实现类比推理的解答者;

 10. planner.py : 负责意图拆解;

 11. model_provider.py : 统一模型调用接口;

 12. virtual_env.py : 提供虚拟环境;

 13. test_virtual.py : 用于自动测试;

 14. utils.py : 用于解析审查者大模型返回回来的东西;

 15. config.py ：全局配置文件;

 16. tokenizer.py ：本地中文分词器;

 17. main.py : 综合所有模块,程序主入口;

## 使用指南

### 下载项目

你可以用 Git 克隆：在终端执行 `git clone https://github.com/DavidDengll/The-new-living-things.git`，然后 `cd The-new-living-things`。或者直接在 GitHub 页面点击绿色 “Code” 按钮下载 ZIP 并解压。

### 安装依赖

在项目文件夹中打开终端，运行 `pip install -r requirements.txt`，等待安装完成。

### 配置模型

打开 `config.py`，设置 `MODEL_NAME`（如 `"glm-4-flash"`）、`API_BASE_URL`（智谱为 `"https://open.bigmodel.cn/api/paas/v4/"`，Ollama 本地为 `"http://localhost:11434/v1/"`）和 `API_KEY`。智谱的 Key 需去官网注册获取。

### 测试模型连接

运行 `python test_provider.py`，如果输出正常回复，说明配置正确。

### 运行系统

键盘交互模式用 `python main.py`，每轮输入场景描述，直接回车使用默认场景。虚拟环境自动测试用 `python test_virtual.py`，会在 5×5 网格中自动运行 100 轮测试。两者都可用 `Ctrl+C` 退出。

### 自定义配置

所有可调参数集中在 `config.py`，包括发言冲动、念头通过线、重试次数、记忆升级条件等。视觉模式可在 `keyboard`、`simulate`、`real` 之间切换。- `USE_CLOUD_ENCODER`：设为 `True` 使用云端大模型编码（效果好但消耗 Token），`False` 使用本地 GPU 编码（免费、快）。

### 关于模型费用

本次测试使用 GLM-4.7-Flash，是智谱 AI 的永久免费模型。如果条件允许，可以把 API 换成 Ollama 等本地模型，实现完全免费的本地闭环。

## 自我优化机制

系统内置了多种优化手段：回复缓存、分类缓存、搜索缓存，思考者附带解答，解答者纯规则触发，置信度客观计算，规划者规则预判，审查缓存，情绪影响发言冲动，反驳冲动，好奇心自然增长，批量语义审查，特征自更新，好奇心提问，类比推理，动作规划与模拟执行，虚拟环境自然交互。这些机制共同使系统在不断运行中自我调整。
