> 一个由**真随机**驱动的实验性意识模拟系统。
> 它不是普通的聊天程序，而是一个拥有“内核冲动”与“外部语言”的双层心智模型。

## 🧠 我为什么这样设计

### 人类思考的秘密
我发现，人类思考的过程并不是直接“想”出答案。我们的大脑会先随机冒出很多乱七八糟的念头（就像神经元在自发放电），然后自己给自己打分，把最合适的那个挑出来。这个过程太快了，所以我们平时根本察觉不到。

我就想：能不能用代码模拟这个过程？

### 1. 真随机驱动 — 模拟大脑噪声
我用操作系统硬件熵（或者量子随机数）来决定：
- 要不要“说话”
- 说几句话
- 每句话多长
然后通过随机 ASCII 码生成最原始的“念头”。它无时无刻都在随机——就跟大脑里的神经噪声一样，是“想法”诞生的土壤。

### 2. 场景分类 — 默认本地分词，可选云端增强
视觉模型看到场景后，默认使用本地分词器（`tokenizer.py`）提取关键词。本地分词基于常见词表和规则，不调用任何 API，零成本。

如果各位有钱想追求更高精度，可以在 `config.py` 里把 `USE_CLOUD_CLASSIFIER` 改成 `True`，系统会调用大模型进行词性标注和主要描述对象识别。分类器做了 JSON 解析容错和自动降级，大模型失败时会自动回退到本地分词。

### 3. 自我审查 — 语义级批量审查官
生成的乱码会交给**审核官**。审核官会调用大模型，根据视觉模型看到的场景和情绪状态，从语义层面批量判断所有念头是否有意义（0~10 分）。审核官会根据视觉关键词从记忆库中检索**相关记忆**（而不是随机抽取），避免无关记忆干扰判断。

**反驳冲动**：当念头连续被否定时，系统会累积反驳冲动。审核官评分时会对边缘分数更宽容，发言概率也会临时提高。

### 4. 情绪模型 — 模拟内在状态
三维情绪系统：精力、好奇心、愉悦。情绪随时间衰减，被事件影响。审核官打出低分时好奇心自然增长。情绪直接影响发言冲动——好奇高更容易开口，精力低更倾向于沉默。

### 5. 记忆系统 — 遗忘与巩固
三层记忆，模仿人脑：
- **短期记忆**：一天删一次
- **长期记忆**：一年删一次
- **永久记忆**：永远不删
一天内被连续提及 N 次 → 短期升长期；一年内被提 N 次 → 长期升永久。所有记忆用 SQLite 存在硬盘上，关机重启不丢失。

**特征自更新**：每次学到新描述时，新旧特征词会被分词并统计历史累积词频，高频词保留，低频词淘汰。特征碎片按词边界取子串，不会跨词取到分隔符。

### 6. 生成策略 — 四层优先级
思考者生成念头时有四层策略（按优先级）：
1. **hint 引导**：审核官反馈的最相关记忆概念作为方向
2. **词素拼图**：从记忆碎片中随机拼接新词
3. **整词插入**：从记忆库取完整片段
4. **兜底随机**：纯随机字母（只在记忆库完全为空时启用）

### 7. 沉默时的内心闪过
当内核决定不说话时，仍会生成简短念头交给审核官判断。有意义就附在回答末尾（如“内心一闪：猫”），无意义则留下低调的“（…）”。沉默时的失败也会被记录，适当调整长期行为倾向。

## 🧱 模块说明
| 文件 | 功能 |
|------|------|
| `randomness.py` | 从操作系统熵池提取真随机数 |
| `memory.py` | 三层记忆系统（SQLite 持久化，特征累积更新，按词边界取碎片） |
| `emotion.py` | 情绪模型（精力/好奇/愉悦，随时间衰减，事件触发，状态保存） |
| `vision.py` | 视觉模型（支持模拟文字/真实图片/键盘输入三种模式） |
| `tokenizer.py` | 本地中文分词器（基于词表+规则，零成本） |
| `classifier.py` | 云端语义分类器（可选，需要时在 config.py 开启） |
| `thinker.py` | 思考者，四层生成策略 |
| `reviewer.py` | 语义审查官，批量审查，基于关键词检索相关记忆 |
| `language_model.py` | 大模型外壳 + 本地兜底输出（空返回自动重试） |
| `main.py` | 主系统，整合所有模块，支持沉默惩罚和反驳冲动 |

## 📖 使用指南

### 环境要求
- Python 3.8+
- 网络可访问 `api.zhipuai.cn`

### 1. 安装
```bash
git clone https://github.com/DavidDengll/The-new-living-things.git
cd The-new-living-things
pip install zhipuai
2. 获取 API Key
访问 智谱AI开放平台 注册并完成实名认证。

在控制台创建 API Key。

打开 config.py，将 Key 填入：

python
ZHIPU_API_KEY = "sk-你的key"
3. 运行
bash
python main.py
运行后每轮会提示输入场景描述，直接回车使用默认场景，输入 Ctrl+C 退出。

4. 自定义配置
所有可调参数集中在 config.py：

参数	默认值	说明
SPEAK_THRESHOLD	0.6	发言冲动（0.2=话痨，0.9=沉默）
MIN_MEANING_SCORE	4	念头通过线（0~10）
MAX_RETRIES	3	重试次数
SHORT_TO_LONG_COUNT	5	短期→长期所需提及次数
LONG_TO_PERMANENT_COUNT	7	长期→永久所需提及次数
视觉模式切换
在 config.py 中修改 VISION_MODE 参数：

模式	值	说明
键盘输入（当前默认）	"keyboard"	每轮手动输入场景描述
模拟场景	"simulate"	使用固定的 SCENE_DESCRIPTION 文字
真实摄像头/图片	"real"	需要填写 IMAGE_PATH，且需要智谱多模态额度
vision.py 里有个 true or false 的摄像头开关，各位可以根据需要调整。config.py 里已经备份了摄像头输入的配置，想用直接改模式就行。

云端分类器（可选）
默认关闭，省钱。想开启的话在 config.py 里改：

python
USE_CLOUD_CLASSIFIER = True
开启后场景分类会更精准（词性标注 + 主语识别），但每次输入会多一次 API 调用。大模型失败时自动回退到本地分词。

5. 各位也可以自己调节想要改的东西
所有参数都在 config.py 里，不用翻源码。记忆升级条件、情绪衰减速度、反驳冲动强度、生成策略概率都可以自己调。

6. 故障排查
现象	可能原因	解决方法
No module named 'zhipuai'	未安装依赖	执行 pip install zhipuai
提示 API Key 无效	Key 未填写或格式错误	检查 config.py 中 Key 是否以 sk- 开头
长时间无响应或超时	网络无法访问智谱 API	检查网络连接，尝试 ping api.zhipuai.cn
大模型回复为空	API 返回空内容	系统会自动重试，连续失败会用本地兜底模板
数据库报错	文件损坏或被占用	关闭其他使用该数据库的程序；如仍不行，备份后删除 entropy_memory.db 重新运行

本地分词默认免费：场景分类走本地词表，不花钱；可选云端增强

情绪影响发言冲动：好奇高更容易开口，精力低更倾向于沉默

反驳冲动：连续被否时反而更想说，审核官也会适当宽容

审查官引导记忆方向：关联概念作为 hint 传给下一轮生成

词素拼图：从记忆碎片中随机拼接新词，按词边界取碎片

动态发言阈值：说得好就多开口，说得差就沉默；沉默失败也会被惩罚

特征自更新：历史累积词频统计，高频词保留，低频词淘汰

好奇心提问：连续遇到陌生事物会主动发问

情绪演化：精力、好奇、愉悦随时间变化

状态持久化：记忆和情绪在关机后不丢失

本地兜底：API 不可用时仍能输出

沉默内心闪过：不说话时内核仍有念头产生

🔬 与现有项目的区别
市面上有使用量子随机数的项目，有三层记忆系统，也有 LLM 自我审查技术。但熵灵是一个将这些技术有机组合的实验性项目：真随机生成原始念头 → 本地分词提取关键词 → 批量语义审查并反馈关联概念 → 记忆碎片拼图引导下一轮生成 → 情绪和反驳冲动影响发言与审查 → 记忆固化 → 自主决定表达（包括沉默时的内心闪过）。
