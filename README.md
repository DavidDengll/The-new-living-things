# 熵灵 (Entropy Sprite)

> ⚠️ 由于本人没钱买显卡，没法装本地大模型。如果各位有钱可以装在本地大模型，就不用烧 Token 了，能实现真正的本地闭环且完全免费。本次测试所使用的模型为 GLM-4.7-Flash，是智谱 AI 的永久免费模型。

## 🧠 我为什么这样设计

### 人类思考的秘密
我发现，人类思考的过程并不是直接"想"出答案。我们的大脑会先随机冒出很多乱七八糟的念头（就像神经元在自发放电），然后自己给自己打分，把最合适的那个挑出来。这个过程太快了，所以我们平时根本察觉不到。

我就想：能不能用代码模拟这个过程？

### 1. 真随机驱动 — 模拟大脑噪声
我用操作系统硬件熵（或者量子随机数）来决定：
- 要不要"说话"
- 说几句话
- 每句话多长
然后通过随机 ASCII 码生成最原始的"念头"。它无时无刻都在随机——就跟大脑里的神经噪声一样，是"想法"诞生的土壤。

### 2. 自我审查 — 语义级批量审查官
生成的乱码会交给**审核官**。审核官会调用大模型，根据视觉模型看到的场景和情绪状态，从语义层面批量判断所有念头是否有意义（0~10 分）。

审核官会根据视觉关键词从记忆库中检索相关记忆，审查结果会被缓存，相同场景+相同念头不重复调用 API，节省 Token。

**反驳冲动**：当念头连续被否定时，系统会累积反驳冲动。审核官评分时会对边缘分数更宽容，发言概率也会临时提高。

### 3. 场景分类 — 云端大模型 + 本地 jieba 兜底
视觉模型看到场景后，优先调用云端大模型进行分词、词性标注和主要描述对象识别。分类结果会被缓存，相同文本不重复分类。如果云端调用失败，自动降级为本地 jieba 分词，保证系统不会崩溃。

### 4. 情绪模型 — 模拟内在状态
三维情绪系统：精力、好奇心、愉悦。情绪随时间衰减，被事件影响。审核官打出低分时好奇心自然增长。情绪直接影响发言冲动——好奇高更容易开口，精力低更倾向于沉默。

### 5. 记忆系统 — 遗忘与巩固
三层记忆，模仿人脑：
- **短期记忆**：一天删一次
- **长期记忆**：一年删一次
- **永久记忆**：永远不删
一天内被连续提及 N 次 → 短期升长期；一年内被提 N 次 → 长期升永久。所有记忆用 SQLite 存在硬盘上，关机重启不丢失。

**联网学习**：思考者遇到不认识的概念时，自动联网搜索（DuckDuckGo），用大模型总结核心特征后存入记忆库。同一关键词一天内不重复搜索，节省 Token。

**特征管理**：新增概念时用 jieba 提取关键词合并特征，避免场景描述碎片污染记忆库。

### 6. 生成策略 — 四层优先级
思考者生成念头时有四层策略（按优先级）：
1. **hint 引导**：审核官反馈的最相关记忆概念作为方向
2. **词素拼图**：从记忆碎片中随机拼接新词
3. **整词插入**：从记忆库取完整片段
4. **兜底随机**：纯随机字母（只在记忆库完全为空时启用）

### 7. 沉默时的内心闪过
当内核决定不说话时，仍会生成简短念头交给审核官判断。有意义就附在回答末尾（如"内心一闪：猫"），无意义则留下低调的"（…）"。沉默时的失败也会被记录，适当调整长期行为倾向。

### 8. 解答者 — 类比推理引擎
新增**解答者**模块。遇到问题时，不只是搜索答案，而是从记忆库中检索相关概念，构建类比链，然后基于类比推理给出答案。如果记忆库知识不足，自动降级为联网搜索补充。

**思考者附带解答**：当遇到不认识的关键词时，思考者联网学习的同时会顺便解答问题，一次 API 调用完成两件事。解答者检测到思考者已附带答案时，自动跳过独立调用，进一步节省 Token。

**触发条件**：纯规则判断（疑问词 + 祈使模式），不调大模型，零 Token 消耗。

**置信度**：基于类比数量客观计算（≥2 条=高，1 条=中，0 条=低），不再依赖大模型自报。

### 9. 规划者 — 意图到动作的拆解
新增**规划者**模块。当大模型回复、思考者插嘴或解答者答案中包含物理动作意图时，自动拆解为具体的动作步骤。

**省钱设计**：先用本地规则检查文本中是否包含物理动作词，没有就直接跳过，完全不调大模型。只有确实包含动作词时才调用大模型做精细拆解。

**模拟执行**：因为没有硬件，规划者会打印动作序列并模拟执行，方便测试整个闭环。等有钱买机械零件，只需替换模拟执行为真实控制 API。

## 🧱 模块说明
| 文件 | 功能 |
|------|------|
| `randomness.py` | 从操作系统熵池提取真随机数 |
| `memory.py` | 三层记忆系统（SQLite 持久化，关键词提取合并，按词边界取碎片） |
| `emotion.py` | 情绪模型（精力/好奇/愉悦，随时间衰减，事件触发，状态保存） |
| `vision.py` | 视觉模型（支持模拟文字/真实图片/键盘输入三种模式） |
| `classifier.py` | 云端语义分类器（分词+词性标注+主语识别，带缓存，失败降级 jieba） |
| `thinker.py` | 思考者，四层生成策略 + 联网学习 + 附带解答 + 搜索缓存 |
| `reviewer.py` | 语义审查官，批量审查，支持缓存和动态 Token |
| `language_model.py` | 大模型外壳 + 回复缓存 + 本地兜底输出 |
| `solver.py` | 解答者，类比推理 + 语义检索 + 置信度客观计算 |
| `planner.py` | 规划者，意图拆解 + 动作排序 + 规则预判 + 模拟执行 |
| `main.py` | 主系统，整合所有模块，思考者与解答者协作省 Token |

## 📖 使用指南

### 环境要求
- Python 3.8+
- 网络可访问 `api.zhipuai.cn`

### 1. 安装
```bash
git clone https://github.com/DavidDengll/The-new-living-things.git
cd The-new-living-things
pip install -r requirements.txt
```

### 2. 获取 API Key

1. 访问 [智谱AI开放平台](https://open.bigmodel.cn/) 注册并完成实名认证。
2. 在控制台创建 API Key。
3. 打开 `config.py`，将 Key 填入：

**python**

```
ZHIPU_API_KEY = "sk-你的key"
```

### 3. 运行

**bash**

```
python main.py
```

运行后每轮会提示输入场景描述，直接回车使用默认场景，输入 `Ctrl+C` 退出。

### 4. 自定义配置

所有可调参数集中在 `config.py`：

| 参数                        | 默认值  | 说明                           |
| --------------------------- | ------- | ------------------------------ |
| `SPEAK_THRESHOLD`         | `0.6` | 发言冲动（0.2=话痨，0.9=沉默） |
| `MIN_MEANING_SCORE`       | `4`   | 念头通过线（0~10）             |
| `MAX_RETRIES`             | `3`   | 重试次数                       |
| `SHORT_TO_LONG_COUNT`     | `5`   | 短期→长期所需提及次数         |
| `LONG_TO_PERMANENT_COUNT` | `7`   | 长期→永久所需提及次数         |

#### 视觉模式切换

在 `config.py` 中修改 `VISION_MODE` 参数：

| 模式                            | 值             | 说明                                          |
| ------------------------------- | -------------- | --------------------------------------------- |
| **键盘输入** （当前默认） | `"keyboard"` | 每轮手动输入场景描述                          |
| **模拟场景**              | `"simulate"` | 使用固定的 `SCENE_DESCRIPTION` 文字         |
| **真实摄像头/图片**       | `"real"`     | 需要填写 `IMAGE_PATH`，且需要智谱多模态额度 |

#### 关于模型费用

本次测试使用的模型为  **GLM-4.7-Flash** ，是智谱 AI 的永久免费模型。只要在免费额度内使用，不会产生费用。如果各位有钱买显卡，可以把代码里的 `zhipuai` 换成 Ollama 等本地模型，就能实现 **完全免费的本地闭环** 。

### 5. 各位也可以自己调节想要改的东西

所有参数都在 `config.py` 里，不用翻源码。记忆升级条件、情绪衰减速度、反驳冲动强度、生成策略概率都可以自己调。

### 6. 故障排查

| 现象                          | 可能原因                 | 解决方法                                                                        |
| ----------------------------- | ------------------------ | ------------------------------------------------------------------------------- |
| `No module named 'zhipuai'` | 未安装依赖               | 执行 `pip install -r requirements.txt`                                        |
| 提示 API Key 无效             | Key 未填写或格式错误     | 检查 `config.py` 中 Key 是否以 `sk-` 开头                                   |
| 长时间无响应或超时            | 网络无法访问智谱 API     | 检查网络连接，尝试 `ping api.zhipuai.cn`                                      |
| 大模型回复为空                | API 返回空内容           | 系统会自动重试，连续失败会用本地兜底模板                                        |
| 数据库报错                    | 文件损坏或被占用         | 关闭其他使用该数据库的程序；如仍不行，备份后删除 `entropy_memory.db` 重新运行 |
| 联网搜索失败                  | 未安装 duckduckgo_search | 执行 `pip install duckduckgo_search`                                          |

## 🔄 自我优化机制

1. **回复缓存** ：相同场景+情绪不重复调大模型，直接复用缓存
2. **分类缓存** ：相同文本不重复分类，失败自动降级 jieba
3. **搜索缓存** ：同一关键词一天内不重复联网搜索
4. **思考者附带解答** ：联网学习时顺便解答，省掉解答者的独立 API 调用
5. **解答者纯规则触发** ：不调大模型判断是否是疑问句
6. **解答者检索无重排序** ：直接取 jieba 匹配结果，不调大模型
7. **置信度客观计算** ：基于类比数量，不依赖大模型自报
8. **规划者规则预判** ：没有物理动作词直接跳过，不调大模型
9. **审查缓存** ：相同场景+相同念头不重复调 API
10. **情绪影响发言冲动** ：好奇高更容易开口，精力低更倾向于沉默
11. **反驳冲动** ：连续被否时反而更想说，审核官也会适当宽容
12. **好奇心自然增长** ：审核官打出低分时好奇心自动上升
13. **批量语义审查** ：一次 API 调用判断所有念头
14. **特征自更新** ：用 jieba 提取关键词合并，避免场景碎片污染
15. **好奇心提问** ：连续遇到陌生事物会主动发问
16. **类比推理** ：解答者基于记忆做类比推理，不是死记硬背
17. **动作规划 + 模拟执行** ：意图自动拆解为动作步骤并模拟执行

## 🔬 与现有项目的区别

市面上有使用量子随机数的项目，有三层记忆系统，也有 LLM 自我审查技术。但熵灵是一个将这些技术有机组合的实验性项目：真随机生成原始念头 → 云端语义分类场景（带缓存） → 批量语义审查并反馈关联概念 → 记忆碎片拼图引导下一轮生成 → 情绪和反驳冲动影响发言与审查 → 记忆固化（关键词提取防污染） → 思考者联网学习+附带解答 → 解答者类比推理 → 规划者意图拆解与模拟执行。
